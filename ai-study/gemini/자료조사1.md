# Gemini 에이전트 및 기능 지원 조사

Claude Code의 Agent, Sub-agent, Skills, Command와 유사한 개념들이 Google Gemini 생태계(주로 Vertex AI Agent Builder 및 Gemini Code Assist)에서도 강력하게 지원되고 있습니다.

## 요약: 기능 비교표

| 개념 (Claude Code) | Gemini / Google Cloud 대응 개념 | 설명 |
| :--- | :--- | :--- |
| **Agent** (에이전트) | **Vertex AI Agents** / **Gemini Agents** (Agent Mode) | Gemini Pro/Flash 모델을 기반으로 추론, 계획, 실행을 수행하는 자율 시스템입니다. IDE에서는 "Agent Mode"로 제공됩니다. |
| **Sub-agent** (서브 에이전트) | **Sub-agents** (in Agent Builder) | 복잡한 작업을 작은 단위로 쪼개어 처리하는 계층적 구조. 메인 에이전트가 서브 에이전트에게 작업을 위임합니다. |
| **Skills** (스킬) | **Tools** (Extensions, Function Calling) | 외부 시스템 연동 도구. Google Search, Code Execution, RAG 등을 포함합니다. |
| **Commands** (커맨드) | **Slash Commands** / **Custom Commands** | `/`로 시작하는 단축 명령어. 내장 명령어(`/fix`, `/test`) 외에 사용자 정의 커맨드도 지원합니다. |

---

## 상세 분석

### 1. 에이전트 (Agents)
Google은 **Vertex AI Agent Builder**와 **Gemini Code Assist**를 통해 에이전트 구축 및 활용을 지원합니다.
- **Agent Mode (Code Assist):** 개발 환경에서 여러 파일에 걸친 복잡한 리팩토링이나 기능 구현을 스스로 계획하고 수행합니다.

### 2. 서브 에이전트 (Sub-agents)
복잡한 워크플로우를 처리하기 위해 메인 에이전트(Orchestrator)가 하위 에이전트를 호출하는 패턴을 공식적으로 지원합니다.
- **Agent2Agent:** 서로 다른 에이전트끼리 통신하며 협업할 수 있는 프로토콜을 제공합니다.

### 3. 스킬/도구 (Skills -> Tools)
Claude의 Skills에 해당하는 기능은 Gemini에서 **Tools** 또는 **Extensions**로 불립니다.
- **Built-in Tools:** Google Search, Code Execution(Python 샌드박스 실행).
- **Function Calling:** 사용자 정의 API 호출 지원.
- **MCP 지원:** 최근 Gemini Code Assist는 **Model Context Protocol (MCP)**를 지원하여, Claude와 마찬가지로 다양한 외부 도구(PostgreSQL, Slack 등)를 에이전트에 쉽게 연결할 수 있습니다.

### 4. 커맨드 (Commands)
Claude Code의 커맨드 기능과 유사한 **Slash Command** 기능을 제공합니다.

*   **Built-in Slash Commands:**
    *   `/generate`: 코드 생성
    *   `/fix`: 버그 수정
    *   `/explain`: 코드 설명
    *   `/test`: 테스트 케이스 생성
    *   `/doc`: 문서화(주석) 생성

*   **Custom Commands:**
    *   사용자가 직접 `.gemini/commands` (또는 설정)에 커맨드를 정의하여 사용할 수 있습니다.
    *   예: `/style` 커맨드를 만들어 "우리 팀 스타일 가이드에 맞춰 포맷팅해"라는 지시를 저장해두고 재사용 가능.

## 결론
Gemini 생태계는 Claude Code의 핵심 기능들(Agent, Skill, Command)을 모두 대응하는 기능(`Agent Mode`, `Tools`, `Slash/Custom Commands`)으로 제공하고 있습니다. 특히 **Custom Command**를 통해 자신만의 워크플로우를 단축키처럼 만들 수 있는 점은 매우 강력합니다.
