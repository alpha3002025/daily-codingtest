# H-Index

## 문제 설명
어떤 과학자의 논문 `n`편 중, `h`번 이상 인용된 논문이 `h`편 이상이고 나머지 논문이 `h`번 이하 인용되었다면 `h`의 최댓값이 이 과학자의 H-Index입니다.
예: `[3, 0, 6, 1, 5]` -> 정렬 `[6, 5, 3, 1, 0]` -> `3`번 이상 인용된 논문이 3편(`6, 5, 3`). H-Index = 3.

### 핵심 개념
1.  **정렬 (Sorting)**: 인용 횟수를 내림차순으로 정렬하면 H-Index를 쉽게 찾을 수 있습니다.
2.  **조건 검사**: 정렬된 리스트에서 인덱스 `i`는 "현재 논문보다 인용 횟수가 많은(또는 같은) 논문의 개수"와 관련이 있습니다.
    - 내림차순 정렬: `citations[i]`는 $i+1$번째로 많이 인용된 논문의 인용 횟수입니다.
    - 이때 `citations[i] >= i + 1` 이라면, $i+1$ 편의 논문이 `citations[i]`번 이상 인용된 것입니다.
    - H-Index는 `citations[i] >= i + 1`을 만족하는 `i + 1`의 최댓값입니다.

## Python 풀이

```python
def solution(citations):
    # 내림차순 정렬: [6, 5, 3, 1, 0]
    citations.sort(reverse=True)
    
    # enumerate로 (순위, 인용횟수) 쌍을 확인
    # i는 0부터 시작하므로 '개수'는 i+1
    for i, citation in enumerate(citations):
        # 인용 횟수(citation)보다 논문 개수(i+1)가 더 커지는 순간
        # 그 직전까지가 H-Index 조건을 만족함.
        # 즉, citation >= i+1 인 마지막 구간의 i+1이 답.
        
        if citation < i + 1:
            return i
            
    # 모든 논문이 (논문 수)번 이상 인용된 경우
    return len(citations)
```

### 코드 설명
- `[6, 5, 3, 1, 0]`
    - i=0: 6 >= 1 (O) -> 현재 H값 후보 1
    - i=1: 5 >= 2 (O) -> 현재 H값 후보 2
    - i=2: 3 >= 3 (O) -> 현재 H값 후보 3
    - i=3: 1 >= 4 (X) -> 여기서 멈춤. 정답은 3.
- 만약 루프가 끝까지 돌면(예: `[100, 100]` -> 2편 모두 2번 이상 인용), 논문의 전체 개수가 H-Index가 됩니다.
