# [PCCE 기출문제] 9번 / 지폐 접기

## 문제 설명
[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/340199)

지갑의 크기(`wallet`)와 지폐의 크기(`bill`)가 주어집니다.
지폐를 지갑에 넣기 위해 다음과 같은 규칙으로 접습니다:
1. 지폐의 긴 쪽 길이가 지갑의 긴 쪽 길이보다 크거나, 지폐의 짧은 쪽 길이가 지갑의 짧은 쪽 길이보다 크다면 지폐를 접어야 합니다.
2. 접을 때는 항상 길이가 긴 쪽을 반으로 접습니다.
3. 접은 후에는 소수점 이하를 버립니다. (`floor` 처리)
4. 지폐를 90도 돌려서 지갑에 들어간다면 멈춥니다.
    - 즉, `(bill[0] <= wallet[0] and bill[1] <= wallet[1])` 또는 `(bill[1] <= wallet[0] and bill[0] <= wallet[1])` 이면 성공입니다.
5. 지갑에 넣을 수 있을 때까지 접은 최소 횟수를 반환하세요.

## 해결 전략
지폐를 접는 과정은 단순히 `긴 쪽 // 2`를 수행하는 반복 작업(Simulation)입니다.
지폐와 지갑의 가로/세로를 비교할 때, **항상 작은 값은 작은 값끼리, 큰 값은 큰 값끼리 비교**하면 회전 여부까지 한 번에 고려할 수 있습니다.
즉, `min(bill) <= min(wallet)` 이고 `max(bill) <= max(wallet)` 이면 들어가는 것입니다.

### 알고리즘 순서
1. `answer` (접은 횟수)를 0으로 초기화합니다.
2. 현재 `bill`이 지갑에 들어가는지 검사합니다.
    - `min(bill) <= min(wallet)` AND `max(bill) <= max(wallet)`
    - 들어간다면 반복을 멈추고 `answer`를 반환합니다.
3. 들어가지 않는다면 접습니다.
    - `bill`의 두 변 중 큰 값을 찾아 반으로 줄입니다 (`// 2`).
    - `answer`를 1 증가시킵니다.
4. 2번 과정으로 돌아갑니다.

## Python 코드

```python
def solution(wallet, bill):
    answer = 0
    
    # 지갑 크기 정렬 (작은 변, 큰 변) - 비교 편의를 위해
    wallet.sort()
    
    while True:
        # 지폐도 정렬해서 비교 (회전 고려 효과)
        bill.sort()
        
        # 지갑에 들어가는지 확인
        # 작은 변은 작은 변끼리, 큰 변은 큰 변끼리 비교
        if bill[0] <= wallet[0] and bill[1] <= wallet[1]:
            break
            
        # 안 들어가면 긴 쪽을 접기
        # bill은 이미 정렬되어 있으므로 bill[1]이 더 크거나 같음
        bill[1] //= 2
        
        answer += 1
        
    return answer
```

## 배운 점 / 팁
- **정렬을 이용한 비교**: 사각형 물체를 다른 사각형 구멍에 넣을 때(회전 가능), 가로/세로를 각각 정렬한 뒤 비교하면 회전된 상태까지 포함하여 간단히 판단할 수 있습니다.
- **반복 검사**: 조건이 만족될 때까지 상태를 변경하며 시뮬레이션하는 `while` 루프 구조의 전형적인 문제입니다.
