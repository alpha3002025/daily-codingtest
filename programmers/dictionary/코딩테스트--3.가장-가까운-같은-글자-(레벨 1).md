프로그래머스 문제는 항상 읽을때 고통스럽다. 막상 풀면 간단한 케이스도 많이 있긴 하다.
answer 를 -1 로 채워두는 아이디어 없이 계속 진행할 경우 즉시 처리하는 방식을 고집하게 되는데 이렇게 처리하게 될 경우 코드의 논리가 모호해진다. 입출력이 10000 으로 제한되어 있기에 메모리 공간을 사용하는 것도 나쁘지 않은 판단이다.

# python
```python
from collections import defaultdict

def solution(s):
    d = defaultdict(lambda: -1)
    answer = [-1] * len(s)
    
    for i,c in enumerate(s):
        if d[c] != -1:
            answer[i] = i - d[c]
        d[c] = i
        
    return answer
```
